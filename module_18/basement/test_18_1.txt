Какой командой можно создать приложение в проекте Django?

Чтобы создать новое приложение в проекте Django, используйте следующую команду в терминале:

python manage.py startapp имя_приложения
Замените имя_приложения на желаемое название Вашего приложения. После выполнения этой команды в Вашем проекте будет создана новая папка с необходимой структурой файлов для приложения.

Чтобы запустить локальный сервер для проекта Django, используйте следующую команду в терминале:

python manage.py runserver
По умолчанию сервер будет запущен на http://127.0.0.1:8000/. Если Вы хотите указать другой порт, Вы можете сделать это следующим образом:

python manage.py runserver 8080
Это запустит сервер на порту 8080.



Какая команда позволяет создать проект Django?

Чтобы создать новый проект Django, используйте следующую команду в терминале:

django-admin startproject имя_проекта
Замените имя_проекта на желаемое имя Вашего проекта.



Какой файл отсутствует в проекте (не в приложении) Django по умолчанию после его создания?

После создания проекта Django по умолчанию отсутствует файл settings.py в корне проекта. Все конфигурации хранятся в директории приложения проекта, которая носит имя проекта (по умолчанию создается папка с названием проекта). В этой папке есть файл settings.py, где Вы можете настроить различные параметры проекта.


В каком файле описывается маршрутизация для пользователя?

В Django маршрутизация для пользователя описывается в файле urls.py. Этот файл содержит список URL-шаблонов, сопоставленных с соответствующими представлениями (views) в проекте.
